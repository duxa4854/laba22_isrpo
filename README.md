# JS Calculator

Простой и функциональный веб-калькулятор, реализованный на чистом JavaScript (ES6+) с использованием объектно-ориентированного подхода. Проект демонстрирует принципы модульности, разделения ответственности (Separation of Concerns) и безопасной обработки пользовательского ввода.

## 📋 Описание проекта

Приложение представляет собой интерфейс классического калькулятора, позволяющий выполнять базовые арифметические операции. Код структурирован таким образом, чтобы логика вычислений была полностью отделена от DOM-манипуляций, что облегчает поддержку, тестирование и потенциальное масштабирование проекта.

## ⚙️ Функциональность

### Поддерживаемые операции

- **Сложение** (`+`)
- **Вычитание** (`-`)
- **Умножение** (`*`)
- **Деление** (`/`)
- **Работа с десятичными дробями** (разделитель — точка `.`)
- **Очистка** текущего выражения (`C`)

### Ограничения

- Ввод ограничен базовыми арифметическими действиями (нет поддержки скобок, процентов или сложных функций).
- Точность вычислений ограничена стандартом IEEE 754 (реализовано программное округление до 8 знаков после запятой для устранения проблем плавающей точки, например, `0.1 + 0.2 = 0.3`).
- Запрещен ввод нескольких операторов подряд (кроме случая замены оператора).
- Запрещен ввод более одной точки в одном числе.

### Обработка ошибок

- **Некорректный синтаксис:** Если выражение заканчивается оператором или содержит недопустимые символы, вычисление блокируется.
- **Математические ошибки:** Результат `Infinity` (деление на ноль) или `NaN` перехватывается и заменяется сообщением "Ошибка".
- **Блокировка ввода:** При возникновении ошибки дальнейший ввод блокируется до нажатия кнопки очистки (`C`).

### Класс Calculator

Основной компонент приложения, инкапсулирующий всю логику работы.

**Основные свойства:**

- `display`: Ссылка на DOM-элемент поля ввода (`<input>`).
- `expression`: Строка, хранящая текущее математическое выражение.
- `hasError`: Булев флаг, указывающий на наличие ошибки в текущем состоянии.

**Ключевые методы:**

- `init()`: Инициализирует приложение, навешивает обработчики событий на кнопки.
- `handleButtonClick(value)`: Центральный маршрутизатор действий. Определяет тип нажатой кнопки (цифра, оператор, равно, очистка) и вызывает соответствующий метод.
- `addToExpression(value)`: Добавляет символ к текущему выражению с предварительной валидацией.
- `calculate()`: Запускает процесс вычисления результата.
- `safeEvaluate(expr)`: Безопасно выполняет строковое выражение.
- `updateDisplay()`: Обновляет значение в поле ввода на основе текущего состояния.
- `clear()`: Сбрасывает состояние калькулятора.

### Схема взаимодействия компонентов

1.  **Пользователь** нажимает кнопку в интерфейсе (`index.html`).
2.  Событие `click` перехватывается в `main.js`, который передает управление экземпляру класса `Calculator`.
3.  Метод `handleButtonClick` определяет действие:
    - Если это цифра/оператор → вызывается `addToExpression`.
    - Если это `=` → вызывается `calculate`.
    - Если это `C` → вызывается `clear`.
4.  После изменения внутреннего состояния (`expression`) вызывается `updateDisplay`, который обновляет UI.

## 🧠 Логика работы

### Последовательность действий при нажатии кнопок

**Ввод цифр и точек:**

- Проверяется, не является ли ввод точки дублирующим в текущем числе (метод `hasDuplicateDot`).
- Символ добавляется в конец строки `expression`.
- Экран обновляется.

**Ввод операторов (`+`, `-`, `*`, `/`):**

- Если выражение пусто, разрешается только минус (для отрицательных чисел).
- Если последний символ уже оператор, он заменяется на новый (например, ввод `+` затем `*` заменит плюс на умножение).
- Иначе оператор добавляется в конец.

**Нажатие равно (`=`):**

- Проверяется валидность всего выражения.
- Выполняется вычисление.
- Результат записывается в `expression`, позволяя использовать его в следующих операциях.

**Нажатие очистки (`C`):**

- Строка `expression` обнуляется.
- Флаг `hasError` сбрасывается.
- На экране отображается `0`.

### Валидация ввода

Валидация происходит в реальном времени при добавлении символов и перед вычислением:

- **Проверка дубликатов операторов:** Метод `isLastCharOperator` смотрит на последний символ. Если это оператор, новый оператор заменяет старый.
- **Проверка точек:** Метод `hasDuplicateDot` разбивает выражение по операторам и проверяет последнюю часть (текущее число) на наличие точки.
- **Финальная валидация:** Перед вычислением `isValidExpression` проверяет, не заканчивается ли выражение оператором и нет ли запрещенных символов.

### Как происходит вычисление

1.  Метод `calculate()` вызывает `safeEvaluate()`.
2.  Для вычисления используется конструктор `new Function('return ' + expr)()`, что является более безопасной альтернативой прямому `eval()`.
3.  Полученный результат проверяется на конечность (`isFinite`).
4.  Результат округляется до 8 знаков после запятой, чтобы избежать артефактов плавающей точки JS.
5.  Если на любом этапе возникает ошибка (синтаксическая или математическая), вызывается `showError()`, который выводит сообщение "Ошибка" и блокирует ввод до сброса.
